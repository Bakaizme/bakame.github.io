Git and Github guide

1.Git and Github overview

Git is a free tool for managing changes in source code to devolop software

Github is a Git hosting tool to store the source code from projects

3.Coding guide

Git config commands (to set identity)

type [git config --global user.name "your github username"]

type [git config --global user.email "your email"]

Git add, commit and push commands (to make changes online, remember to save your work before you type these commands)

type [Git add .] -> [Git commit "message"] (it will show the changes) -> [Git push] (the changes are made in your online repository)

Git pull command (to solve conflict)

Conflict happen when changes made in the same file by 2 or more other people and you have to decide which to keep or remove for the final project

Conflict occur -> type [Git pull] (download repository to your local, git fetch) -> Conflict detected -> Decide the changes -> The repository will change base on your decision (git merge)

Git brand command (to make change without ruining your main work)

Type [Git branch "name"] to make a branch -> type [Git check out "name"] to move between branches and main file
